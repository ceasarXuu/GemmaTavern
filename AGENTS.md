# 基本原则
- 最小化提交原则：每进行最小主体改动，要积极提交commit 并写清楚详细的改动说明，无需询问用户即可提交。

- 完成改动后执行真机覆盖安装验证后再结束任务。

# 外部项目参考
## SillyTavern 
本机独立仓库，按仓库名 SillyTavern 访问；不要在公开文档中记录绝对路径。


# 系统约束
- 本项目支持多语言，任何 UI 改动涉及文字要考虑多语言开发。
- Agent / tool 架构中，禁止使用基于关键词、正则、枚举意图的规则来决定是否触发工具调用。
- 工具是否调用必须由 agent 基于完整上下文自行决策；宿主层只允许负责能力注册、权限确认、安全边界、超时、重试、日志与结果注入，不得接管工具触发决策。
